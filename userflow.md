### **1. User Flow (มุมมองของผู้ใช้งาน)**
เน้นที่ประสบการณ์ของนักท่องเที่ยว ตั้งแต่เริ่มเปิดแอปจนได้แผนการเที่ยว

1.  **Onboarding (Get Started):** 
    *   ผู้ใช้เปิดแอป (ไม่ต้อง Login เพื่อความไว) เลือกพิกัดที่อยู่ตอนนี้ (เช่น กรุงเทพฯ, เชียงใหม่) หรือระบุย่านที่อยากไป
2.  **The Swiping (Matching):** 
    *   เข้าสู่หน้าไพ่ (Cards) แสดงรูป **อาหารชื่อดัง** สลับกับ **จุดท่องเที่ยวท้องถิ่น** 
    *   **Action:** ปัดขวา (ชอบ/อยากกิน) | ปัดซ้าย (ไม่สนใจ)
    *   *System Logic:* เมื่อปัดขวาครบจำนวนที่กำหนด (เช่น อาหาร 3 อย่าง + ที่เที่ยว 2 แห่ง) ระบบจะหยุดการปัด
3.  **Result Generation:** 
    *   แอปแสดงหน้า "It's a Match!" พร้อมสรุปรายการที่ผู้ใช้เลือก
4.  **The Itinerary (Final Plan):** 
    *   ระบบจัดลำดับแผนการเดินทาง 1 วัน (เช่น มื้อเช้า -> ที่เที่ยว 1 -> มื้อเที่ยง -> ที่เที่ยว 2) พร้อมปุ่มเชื่อมต่อไปยัง Google Maps

### **2. Process Flow (การทำงานหลังบ้าน/ตรรกะระบบ)**
เน้นการไหลของข้อมูล เพื่อให้ Dev ในทีมเห็นภาพว่าต้องเขียนโปรแกรมอย่างไร

1.  **Data Ingestion:**
    *   เตรียมชุดรูปภาพอาหารและสถานที่ (Database/Mock Data) โดยต้องมี Tag กำกับ (เช่น ย่าน: เยาวราช, ประเภท: Street Food, สไตล์: วัด/ประวัติศาสตร์)
2.  **Randomized Display:**
    *   ดึงรูปภาพจากคลังออกมาแสดงแบบสุ่มตามพิกัดที่ผู้ใช้เลือกในตอนแรก
3.  **Preference Collection:**
    *   เก็บค่า (ID) ของรูปภาพที่ผู้ใช้ปัดขวาลงใน Array
4.  **AI/Algorithm Selection:**
    *   ระบบนำ ID ที่ชอบมาวิเคราะห์หาจุดที่ใกล้กันที่สุด (Geographic Clustering) เพื่อไม่ให้ทริปเดินทางไกลเกินไป 
    *   *Hackathon Shortcut:* ใช้ AI (เช่น Gemini API) ช่วยสรุปลำดับการเดินทางจากลิสต์ที่เลือก
5.  **Output API:**
    *   ส่งแผนการเดินทางกลับไปแสดงผลที่หน้า UI ในรูปแบบ Timeline
